# 发布说明（2026-04-06）

本次版本围绕“文章生成质量、发布可用性、控制台可理解性”做了一轮完整升级。系统现在支持在控制台直接输入单篇链接，跳过热点筛选，沿用既有事实抽取、写作、配图和公众号发稿链路一键处理；同时补上了联网信息自然融入正文、英文小标题本地化、代码块保真、正文插图与段落联动等能力，让生成结果更像可发布文章，而不是模型拼接稿。

## 核心升级

- 新增“指定链接直跑”能力，可直接处理单篇文章并进入完整工作流。
- 联网补充信息不再单独落成“相关阅读”，而是自然融入正文。
- 英文小标题会自动本地化，中文技术文章可读性更稳定。
- 代码块、命令块与交互示例会尽量按原文格式保留，降低技术内容失真。

## 视觉与发布改进

视觉与发布链路做了重点修复。封面改为先按主题生成，再叠加标题；正文配图支持结构化技术图与内容型海报两种方向，并优化了图片尺寸、文字遮挡、插图插入时机与微信兼容性。针对公众号端，又修复了代码块在微信里挤成一行的问题，改进了图片上传方式和大图发稿限制处理。

## 控制台体验优化

- “任务触发”与“任务详情”中的步骤名已统一改为中文显示。
- 失败步骤、当前步骤和运行列表中的步骤信息都做了中文化。
- Token 概览仍保留模型、角色、费用等统计，但展示更贴近实际运行情况。

## 选题与统计修复

选题与统计可靠性也做了增强。旧文章误判为“最新”的问题已经处理，`html_list` 源会补抓详情页标题和发布时间，并支持从正文中提取像 `2025年1月23日` 这样的可见日期；Token 概览则修正了统计口径，空模型脏数据和 mock 调用不会再污染真实计费结果。

## 适合关注的变化点

- 如果你经常手动指定稿源，现在可以直接在控制台输入链接处理。
- 如果你主要看微信成稿效果，这一版值得重点关注代码块、配图和标题修复。
- 如果你依赖 Token 面板评估成本，这一版的统计结果会比之前更干净。
