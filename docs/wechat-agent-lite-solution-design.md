# wechat-agent-lite 整体方案设计

版本：v1.3（最终讨论定稿）  
适用环境：Ubuntu 2核 / 2G，50G硬盘（项目可用10G）  
时区：Asia/Shanghai

## 1. 项目目标与边界

- 每天自动抓取资讯并生成 1 篇微信公众号草稿。
- 架构采用“工作流骨架 + 受约束 Agent 决策层（Constrained Agent）”。
- 控制台不对公网开放，只通过 SSH 隧道访问。
- 仅发送日报邮件，不发送即时失败告警邮件。

## 2. 核心架构

- Web Console：配置管理、任务查看、手动干预、监控看板。
- Orchestrator：状态机调度、重试执行、决策落地、日志审计。
- Model Gateway：统一模型调用、Token 与耗时统计。
- Collector：RSS 与 GitHub 热点抓取（复用原项目来源清单）。
- WeChat Publisher：微信公众号草稿发布。
- SMTP Notifier：日报发送。
- Proxy Adapter：Xray / VLESS 代理接入与健康检查。
- Storage：SQLite + 文件归档 + 10G 配额管理。

## 3. 调度时间与主流程

- 08:30：先做代理与抓取源健康复检。
- 09:00：执行主任务。
- 主任务状态机：
  HEALTH_CHECK -> FETCH -> DEDUP -> RULE_SCORE -> RERANK -> SELECT -> WRITE -> QUALITY_CHECK -> COVER_5D -> COVER_GEN -> COVER_CHECK -> WECHAT_DRAFT -> REPORT_MAIL -> DONE
- 每一步都记录：开始时间、结束时间、耗时、状态、重试次数、错误信息、Token 用量。

## 4. 排序与产出策略

- TopN = 10
- TopK = 8
- 每轮最终只产出 1 篇文章
- 选题融合分：final_score = 0.6 * rule_score + 0.4 * llm_score

## 5. 模型角色与分工（全部可配置）

- decision_llm：流程决策（续跑 / 重跑 / 重生成）
- rerank_llm：TopK 语义重排
- writer_llm：正文写作
- cover_prompt_llm：封面提示词与 5 维解析
- cover_image_model：封面出图

说明：决策模型只输出“动作指令”，不直接调用其他模型。实际执行由 Orchestrator 完成，并受规则闸门约束。

## 6. 重试与退避策略

- 抓取阶段：重试 2 次，退避 5s -> 15s
- 生成阶段：重试 2 次，退避 10s -> 30s
- 发布阶段：重试 3 次，退避 15s -> 45s -> 120s

## 7. 质量闸门策略

- 低于阈值则重生成，最多 3 轮。
- 第 3 轮后仍不达标，选最高分版本发送。
- 日报邮件必须明确标注：
  - 未达阈值但已采用最高分版本
  - 各轮分数与最终采用轮次

## 8. 封面策略

- 封面 5 维：主题主体、场景构图、视觉风格、色彩光线、文案层级。
- 支持控制台操作：
  - 重新生成封面
  - 导入本地图片覆盖
- 全程保留封面版本记录与评分记录。

## 9. 控制台页面清单

- 初始化向导：模型配置、微信配置、SMTP 配置、代理配置
- 设置中心：所有凭据可修改，支持连接测试
- 任务中心：失败步骤定位、错误分层、重试记录、一键重跑
- 质量中心：文章 / 选题 / 封面 / 系统四层评分
- Token 中心：按模型角色与步骤统计
- 性能中心：步骤耗时、P50 / P90 / P95
- 存储中心：10G 配额占用、分项趋势、阈值告警与清理记录

## 10. 手动干预动作

- 重跑本轮（全流程）
- 从失败步骤续跑
- 仅重生成文章
- 仅重生成封面
- 发布失败时保留本地成稿与封面，状态标记“待人工补发”

## 11. 邮件策略（详细日报）

- 每天只发送 1 封日报邮件。
- 日报包含：
  - 任务概览（run_id、时间、状态）
  - 文章标题、摘要、草稿信息
  - 封面 5 维评分与最终采用版本
  - TopN / TopK 摘要
  - 质量评分明细
  - Token 分模型统计
  - 关键步骤耗时明细
  - 失败日志明细
- 邮件里的控制台链接使用 SSH 隧道地址：
  - http://127.0.0.1:18080

## 12. 代理与网络策略

- 使用 Xray / VLESS，支持粘贴分享链接或 JSON 配置。
- 代理为运行时配置，不写入代码仓库。
- 08:30 健康检查先测代理可用性，再测抓取源可达性。

## 13. 存储与生命周期（10G）

- 分层保留：
  - 核心数据：365 天
  - Token / 步骤耗时明细：90 天
  - 原始抓取缓存：30 天
- 阈值清理：
  - >85% 触发预警清理
  - >95% 触发紧急清理

## 14. 部署与访问

- 部署方式：systemd
- 控制台端口：8080
- SSH 端口：22（默认）
- 本地转发端口：18080
- 控制台不设密码，仅通过 SSH 隧道访问

## 15. 安全与配置管理

- 微信公众号凭据、百炼模型凭据、SMTP 参数、代理参数全部运行时填写。
- 敏感字段加密存储，前端脱敏展示。
- 真实配置、数据库、日志不纳入 Git 仓库。
